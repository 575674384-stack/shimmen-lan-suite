# 内网协同工具套件 — Spec Brief

## 1. 项目概述

为小型办公团队打造的纯内网去中心化协同工具。运行在同网段（局域网）下，无需服务器、无需登录，各节点完全对等。提供文件传输、共享文件夹同步、团队看板、共享信息栏等办公高频功能。

## 2. 核心功能清单

### F1. 内网用户发现
- 启动后自动通过 UDP 广播/mDNS 扫描同网段在线用户
- 用户列表显示：自定义用户名 + IP 地址
- 无登录机制，首次启动时让用户设置用户名即可
- 心跳保活，离线用户自动从列表移除

### F2. 文件传输窗口（点对点）
- 点击用户 → 打开独立传输窗口（类似微信私聊窗口）
- 支持发送文件、文件夹、图片
- 拖拽文件到窗口即可发送
- 传输进度显示、断点续传
- 传输记录保存在本地 SQLite

### F3. 共享聊天窗（全员群聊）
- 所有在线用户共享一个聊天窗口
- 支持发送文字、文件、图片
- **全员可清屏**（一键清空所有历史，各客户端同步执行）
- 聊天记录不持久化（清屏即消失，或仅保留最近 100 条）
- 新上线用户看不到历史记录（纯实时）

### F4. 共享文件夹（双向同步）
- 每个用户可指定本地任意路径为"共享文件夹"
- 其他用户可浏览所有在线用户的共享文件夹列表
- 订阅后，该文件夹内容双向同步到本地指定路径
- 订阅用户可对文件夹内文件进行编辑/删除/新增，修改自动同步回源和其他订阅者
- 文件冲突处理：最后写入胜出 + 保留冲突副本
- 文件修改历史记录（谁在什么时间改了什么）
- **15 天内文件历史快照**（旧版本自动归档到本地隐藏目录，15 天后自动清理）
- 文件同步采用块级增量同步（大文件只传差异块）

### F5. 共享密码栏
- 所有用户可编辑的共享表格/列表
- 字段：名称 / 账号 / 密码 / 备注
- 去中心化同步，所有节点保持一致
- 采用 CRDT/向量时钟解决并发编辑冲突
- 数据本地加密存储（AES-256，密钥派生自机器特征，防本地明文泄露）

### F6. 共享公告栏
- 所有用户可编辑的富文本公告
- 支持多条公告，可置顶
- 去中心化同步，冲突处理同 F5
- 新上线用户自动获取当前最新公告内容

### F7. 个人工作进度计划栏
- 看板视图：三列（待处理 / 处理中 / 已完成）
- 任务字段：项目名称、截止日期、联系人、优先级、描述
- 鼠标拖拽任务卡片变更状态
- 任务完成后可上传关联文件
- 完成后可选择"归档到共享文件夹"（自动在共享空间创建项目文件夹）
- 数据保存在本地 SQLite

### F8. 团队协作工作进度计划栏
- 基于 F7 的团队视角
- 显示所有用户标记为"对团队可见"的任务
- 支持按"项目"分组的看板视图
- 支持按负责人筛选
- **认领机制**：团队看板中的未分配任务，任何人可点击"认领"，自动复制到个人看板并关联原任务
- 任务状态变更实时同步给所有在线用户
- 团队任务完成后，创建者可将任务归档到共享文件夹

## 3. 技术栈

| 层级 | 选型 | 理由 |
|---|---|---|
| 应用壳 | Tauri v2 | 轻量（<15MB），Windows 自带 WebView2，系统托盘原生支持 |
| 前端 UI | React 18 + TypeScript + Tailwind CSS | 微信风格 IM 界面成熟生态，开发效率高 |
| 图标库 | lucide-react | 600+ 精美 SVG 图标，风格统一 |
| 看板拖拽 | @dnd-kit/core | React 生态最成熟的拖拽库 |
| 后端/系统 | Rust | Tauri 内置，高性能处理网络、文件、系统调用 |
| 本地数据库 | SQLite (rusqlite) | 零配置，存储用户列表、任务、传输记录、文件索引 |
| 网络发现 | UDP 广播 + 自定义心跳协议 | 简单可靠，同网段发现 |
| P2P 传输 | TCP 直连 | 局域网内无需打洞，延迟最低 |
| 共享状态同步 | 向量时钟 + LWW (Last-Write-Wins) | 足够解决去中心化冲突 |
| 文件同步 | rsync 算法简化版（块级哈希比对） | 大文件增量同步，节省带宽 |

### 通信模型

- **发现层**：UDP 广播（端口固定，如 23333），每秒心跳，离线 3 秒未收到心跳即标记离线
- **控制层**：TCP 长连接（端口如 23334），用于状态同步、聊天、看板更新、文件传输信令
- **数据层**：文件传输另开 TCP 直连（动态端口或复用控制连接），大文件分块传输

### 去中心化设计

- **无 Master 节点**：所有节点权力完全一致
- **无登录/认证**：用户名仅作展示，IP 是身份标识
- **状态一致性**：共享状态（密码栏、公告栏、团队看板）通过向量时钟 + LWW 解决冲突
- **新节点加入**：自动向所有在线节点广播"我来了"，并请求当前共享状态快照

## 5. 数据模型（核心）

### 用户 (User)
```
id: string (UUID)
username: string
ip: string
last_seen: timestamp
status: online | offline
```

### 消息 (ChatMessage) — 仅内存/临时存储
```
id: string
sender_id: string
sender_name: string
type: text | file | image
content: string (text 或文件路径/URL)
timestamp: timestamp
```

### 共享文件夹 (SharedFolder)
```
id: string
owner_id: string
owner_name: string
local_path: string (本机绝对路径)
name: string (展示名)
subscribers: User[] (谁订阅了)
sync_status: syncing | paused | error
```

### 文件版本 (FileVersion) — 用于历史快照
```
file_id: string (hash of path)
shared_folder_id: string
version_id: int
file_hash: string (blake3)
file_size: int
modified_at: timestamp
modified_by: string (user_id)
snapshot_path: string (本地隐藏目录中的路径)
```

### 任务 (Task)
```
id: string
title: string
project: string
deadline: date | null
contact: string
priority: low | medium | high
description: string
status: todo | doing | done
creator_id: string
assignee_id: string | null
is_team_visible: boolean
attached_files: string[] (本地路径)
archived_to_folder_id: string | null
created_at: timestamp
updated_at: timestamp
version: vector_clock
```

### 密码条目 (PasswordEntry)
```
id: string
name: string
account: string
password: string (AES-256 加密存储)
note: string
created_by: string
updated_at: timestamp
version: vector_clock
```

### 公告 (Announcement)
```
id: string
title: string
content: string (markdown)
is_pinned: boolean
created_by: string
updated_at: timestamp
version: vector_clock
```

- **左侧边栏**：窄条图标导航，鼠标悬停显示 tooltip，选中高亮
- **用户列表**：中间区域显示所有在线用户卡片（头像+用户名+IP），点击用户 → 右侧滑出/弹出文件传输窗口
- **传输窗口**：独立浮动窗口或右侧分栏，聊天记录式显示，底部输入框+拖拽区域
- **看板**：三列横向布局，卡片可拖拽，卡片显示项目名称/截止日期/联系人
- **共享文件夹**：左侧树形目录（所有用户的共享文件夹），右侧文件浏览器，支持拖拽上传

### 系统托盘
- 最小化后缩到右下角托盘
- 左键单击：显示/隐藏主窗口
- 右键菜单：显示窗口 / 设置 / 退出
- 托盘图标动态：有未读消息时显示红点角标

## 7. 非目标（明确不做）

- **远程桌面**：V2 功能，V1 不做
- **互联网穿透/外网访问**：仅支持同网段局域网
- **用户权限系统**：所有用户完全对等，无管理员/访客区分
- **登录/注册/密码验证**：用户名仅作展示，无认证
- **消息持久化/离线消息**：聊天窗清屏即消失，不上线时收不到消息
- **移动端支持**：仅 Windows 桌面端
- **端到端加密**：内网环境暂不做 E2EE，传输明文（后续可加 TLS）

## 8. 验收标准

### Must Have（必须有）
- [ ] 启动后 3 秒内发现同网段其他用户
- [ ] 文件传输速度达到局域网带宽上限的 80% 以上
- [ ] 共享文件夹双向同步延迟 < 3 秒（局域网内）
- [ ] 看板拖拽流畅，状态变更实时同步到其他用户
- [ ] 共享密码栏/公告栏在所有节点保持一致（并发编辑无丢失）
- [ ] 15 天文件历史快照可正常回滚
- [ ] 最小化到托盘，托盘图标可唤醒主窗口
- [ ] 单文件安装包，Windows 10/11 双击即用

### Nice to Have（加分项）
- [ ] 文件传输支持断点续传
- [ ] 共享文件夹支持"仅下载不同步上传"模式
- [ ] 看板支持搜索/过滤
- [ ] 聊天窗支持撤回消息

## 9. 风险与注意事项

| 风险 | 缓解措施 |
|---|---|
| Windows 防火墙拦截 UDP/TCP | 首次启动提示用户添加防火墙例外 |
| 多用户同时修改共享文件夹同一文件 | 冲突副本机制 + 历史快照回滚 |
| 共享密码栏明文存储风险 | 本地 AES-256 加密，内存中解密 |
| 大文件同步占用带宽 | 块级增量同步 + 可配置限速 |
| 向量时钟漂移导致状态不一致 | 定期全量状态同步作为保底 |

## 10. ADR 信号（后续需补充架构决策记录）

- **ADR-1**：去中心化状态同步协议选择（向量时钟 + LWW vs CRDT）
- **ADR-2**：文件同步算法选择（rsync 简化版 vs 整文件比对）
- **ADR-3**：共享密码栏本地加密方案（密钥派生策略）
- **ADR-4**：网络发现协议（UDP 广播 vs mDNS）
